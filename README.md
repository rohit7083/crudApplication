# User Management System

A React-based user management application with CRUD operations, form validation, and toast notifications.

## Setup Instructions

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Start the JSON Server (backend):**
   ```bash
   npx json-server --watch db.json --port 3001
   ```

3. **Start the development server:**
   ```bash
   npm run dev
   ```

4. **Access the application:**
   Open your browser and navigate to `http://localhost:5173`

## How to Add New Fields to the Form

To add new fields to the user form, simply update the `src/config/userSchema.js` file:

```javascript
export const userSchema = [
  // Existing fields...
  {
    name: "fieldName",           // Field identifier
    label: "Field Label",        // Display label
    type: "text",                // Input type (text, email, tel, etc.)
    required: "Error message",   // Validation message
    pattern: {                   // Optional regex validation
      value: /regex/,
      message: "Pattern error message"
    }
  }
];
```

The form will automatically:
- Render the new field
- Apply validation rules
- Display in the table
- Include in create/update operations

## Design Decisions & Assumptions

### Architecture
- **Component-based structure:** Separation of concerns with reusable components
- **Centralized schema:** Single source of truth for form fields in `userSchema.js`
- **API layer:** Isolated API calls in `src/api/users.js`

### Technology Stack
- **React Hook Form:** Efficient form handling with minimal re-renders
- **Axios:** HTTP client for API requests
- **React Toastify:** User-friendly notifications
- **Tailwind CSS:** Utility-first styling
- **JSON Server:** Mock REST API for development

### Key Features
- Dynamic form generation from schema
- Form reset after successful creation
- Dynamic titles (Create/Update)
- Toast notifications for all operations
- Client-side validation with error messages
- Loading states during async operations

### Assumptions
- JSON Server runs on port 3001
- All fields defined in schema are required by default
- Phone number must be exactly 10 digits
- Email validation handled by HTML5 input type
- Users have unique IDs generated by JSON Server
